# Input Checklist

Collect these fields from the user before running a new project deployment.

## Site Identity

- Website positioning: location + industry, for example `贵阳定制`.
- Company name.
- Phone number.
- Target domain or subdomain.
- Logo file path.
- Favicon file path if different.
- WeChat QR code file path if needed.

## Content Automation

- Visible guide/blog frequency: daily, weekly, or custom.
- Hidden GEO/SEO frequency: daily, weekly, or custom.
- Prompt file paths.
- Prompt type labels if alternating, for example `geo` and `geo-with-english`.
- Title/keyword file path with one title per line.
- Whether hidden pages should be indexed or only accessible by URL.

## Accounts And Secrets

- GitHub repository URL.
- GitHub token with Contents read/write and Actions read/write.
- DeepSeek API key.
- Cloudflare API token.
- Cloudflare account email if useful for identifying the account.
- Cloudflare root zone, for example `gz-axy.com`.
- Cloudflare target domain, for example `sjt.gz-axy.com`.
- R2 bucket name if using R2.
- R2 access key ID.
- R2 secret access key.
- R2 endpoint URL.

## Verification Targets

- Production homepage URL.
- Visible blog/list URL.
- Hidden GEO list URL if created.
- Contact/booking page URL.
